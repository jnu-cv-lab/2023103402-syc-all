# 计算机视觉实验六：基于 OpenCV 的局部特征检测、描述与图像匹配

## 实验环境
- 操作系统：Ubuntu / WSL2
- 开发工具：VS Code
- 编程语言：Python 3
- 依赖库：OpenCV (含 contrib，用于 SIFT) + NumPy + Matplotlib

## 实验目的
1. 掌握 ORB 局部特征检测与二进制描述子的原理与使用
2. 掌握基于 BFMatcher + Hamming 距离的特征匹配，以及 crossCheck 的作用
3. 理解 RANSAC 在剔除错误匹配中的几何一致性思想
4. 利用 Homography 完成模板图到场景图的目标定位
5. 通过参数对比实验，分析 `nfeatures` 对匹配数量与内点比例的影响
6. 选做：完成 SIFT + KNN + Lowe ratio test 的对比实验

### 任务 4：目标定位
将 box.png 的四个角点经 Homography 投影至场景图，使用 `cv2.polylines` 绘出红色四边形边框。**定位成功**：四边形位置与场景中真实物体高度吻合。

### 任务 6：参数对比实验

| nfeatures | 模板图关键点 | 场景图关键点 | 匹配数量 | RANSAC 内点 | 内点比例 | 是否成功定位 |
|-----------|--------------|--------------|----------|-------------|----------|--------------|
| 500       | 453          | 500          | 149      | 32          | 0.2148   | 是           |
| 1000      | 865          | 1000         | 287      | 52          | 0.1812   | 是           |
| 2000      | 1589         | 1999         | 511      | 66          | 0.1292   | 是           |

**分析：**
- `nfeatures` 越大，模板图与场景图的关键点数量及总匹配数量都近似线性增长。
- 内点数量随 `nfeatures` 增长而增加（32→52→66），但增速远慢于匹配数量。
- **内点比例反而下降**（0.2148→0.1812→0.1292）。原因是新增的关键点多落在响应较弱、纹理重复或边界模糊的区域，产生了大量"看起来相似但几何不一致"的错误匹配。
- 三组参数都成功定位，说明只要内点的绝对数量足以稳定估计 Homography（≥4 即可，通常 ≥10 已较稳健），定位就能成功。
- **结论：特征点不是越多越好。** 数量过少时内点不足、几何估计不稳；数量过多时引入大量噪声匹配，反而压低内点比例并增加 RANSAC 的计算量。`nfeatures=1000` 是较为均衡的选择。

### 选做任务：SIFT 对比实验

SIFT 检测：box.png 关键点 604，box_in_scene.png 关键点 969，描述子维度 (N, 128)。
匹配采用 KNN(k=2) + Lowe ratio test (阈值 0.75)。

| 方法 | 匹配数量 | RANSAC 内点 | 内点比例 | 是否成功定位 | 运行速度（主观） |
|------|----------|-------------|----------|--------------|------------------|
| ORB  | 287      | 52          | 0.1812   | 是           | 快（二进制描述子 + Hamming 距离）|
| SIFT | 80       | 75          | **0.9375** | 是         | 较慢（浮点 128 维 + L2）|

**对比分析：**
- SIFT 经 Lowe ratio test 后保留的匹配数虽较少（80 < 287），但**几乎全部是正确匹配**，内点比例高达 0.94。
- ORB 不做 ratio test 而采用 crossCheck，对错误匹配的过滤较弱，内点比例低（0.18），但绝对内点数量足以支撑 RANSAC。
- 两种方法在本实验中都成功完成了目标定位，**SIFT 的匹配质量更高、Homography 估计更稳定**；ORB 速度上有优势，适合实时场景。

## 五、必须回答的问题

### 问题 1：什么是特征点？
**结合 box.png 的实验结果：**
- 关键点主要分布在 **包装盒上的文字、图案边缘、商标的角点、纸盒折痕** 等位置。
- 文字、角点、纹理丰富区域在 x、y 两个方向上像素灰度都有显著变化（即图像梯度大、二阶矩矩阵两个特征值都大），在不同视角/光照下仍能稳定出现，因此**可重复性强**。
- 平坦区域（盒子的纯色面、桌面背景）灰度几乎恒定，任意位置都看起来一样，特征"不显著"，无法在不同视图中稳定对应——所以 ORB 几乎不会在这些区域产生关键点。

### 问题 2：什么是特征描述子？
- **关键点**只是图像中的一个位置（坐标 + 尺度 + 方向），告诉我们"这里有一个值得关注的点"。
- **描述子**是一个固定长度的向量，编码了关键点周围邻域的灰度结构，是用于"两个关键点是否是同一物点"的指纹。
- 只有位置无法判断对应关系——两幅图各有上千个关键点，必须借助描述子之间的距离才能匹配。
- ORB 的描述子是 BRIEF 的旋转修正版（rBRIEF），输出 **32 字节 = 256 位**；之所以是二进制描述子，是因为它通过对若干对采样点做灰度比较（>则置 1，否则置 0）得到，每一位就是一次比较结果，紧凑、计算快、占用内存小。

### 问题 3：为什么 ORB 使用 Hamming distance？
- ORB 的描述子是 256 位二进制串。
- **Hamming 距离**衡量两个二进制串中 **不同位的个数**（异或后置 1 位数）。
- 不使用欧氏距离的原因：二进制位上"3-1=2"这种数值差异没有意义，欧氏距离需要乘法/平方/开方等浮点运算；而 Hamming 距离可以用一条 `XOR + popcount` CPU 指令完成，**速度比浮点 L2 快一到两个数量级**，而且在数学上正是二进制空间下的自然度量。

### 问题 4：ORB 为什么对旋转、平移和一定尺度变化具有鲁棒性？
- **平移鲁棒性**：描述子只依赖关键点局部邻域的相对像素结构，不依赖关键点在整图中的绝对位置。物体移动时邻域内容不变，描述子也不变。
- **旋转鲁棒性**：ORB 通过强度质心法（intensity centroid）为每个关键点估计一个主方向 θ，然后将 BRIEF 采样模板按 θ 旋转再做比较——同一关键点旋转后描述子保持一致（即 steered BRIEF）。
- **尺度鲁棒性**：ORB 在多层图像金字塔上检测 FAST 关键点；不同尺度的同一物点会在金字塔的对应层被检测到，从而对一定范围内的缩放保持鲁棒。
- **不能完全抗透视变换**：ORB 假设邻域是相似变换（旋转 + 缩放 + 平移），但透视变换下邻域会发生剪切、形变，比较点的几何关系被打乱。大角度透视下 ORB 描述子会失效——这正是后续要靠 RANSAC + Homography 在全局上修正的原因。

### 问题 5：为什么初始匹配中会有错误匹配？
**结合 ORB 初始匹配图：**
- **错误匹配通常出现在**：重复纹理（如等距条纹）、相似图案（同一字母在不同位置）、低对比度区域、背景干扰、以及描述子局部相似但几何上不对应的点。
- 重复纹理和相似图案让多个关键点的描述子相互之间距离都很近，BFMatcher 取到的"最近邻"未必是真正对应的那个。
- **匹配距离小不一定正确**：距离只衡量描述子在特征空间的相似性，不考虑几何一致性。两个不相关位置只要邻域纹理巧合相似，距离也会很小，但它们在两幅图之间根本不存在物体对应关系。

### 问题 6：RANSAC 的作用是什么？
- RANSAC 通过**随机抽取 4 对匹配点估计候选 Homography → 在该模型下统计有多少对匹配的重投影误差小于阈值（内点）→ 多次迭代后取内点最多的模型**，从而剔除错误匹配。
- RANSAC 的依据是**几何一致性**（描述子相似度只在生成初始匹配时使用过）：所有正确匹配应当满足同一个 Homography。
- **inlier**：满足该 Homography 模型、重投影误差低于阈值（如 5 px）的匹配。
- **outlier**：与该模型不一致、重投影误差大的错误匹配。
- RANSAC 后的匹配线明显更合理，因为它们集中指向同一个物体区域、方向一致；而 outlier（连接到背景或其他位置的乱线）已经被滤除。

### 问题 7：Homography 的意义是什么？
- Homography（单应矩阵）是 3×3 的射影变换矩阵，描述**两幅图像中同一平面物体之间的对应关系**——即"从一个平面经过任意视角看这个平面"对应的像素映射。
- box.png 是包装盒的平面正面图，box_in_scene.png 中盒子表面在另一个视角下也仍是一个平面，所以两图之间的对应是 Homography。
- Homography **只严格成立于平面物体或纯相机旋转**两种情形。
- 如果场景中物体不是平面（如带厚度的立体物），不同深度的点对应不同的视差，单一 Homography 无法同时拟合，会出现：**部分区域对齐、部分区域偏移、定位边框扭曲或穿过物体内部**等问题。

### 问题 8：SIFT 和 ORB 有什么区别？
- **结构差异**：ORB 描述子是 256 位二进制串；SIFT 描述子是 128 维浮点向量（基于 4×4 网格、8 方向梯度直方图）。
- **距离度量**：ORB 用 Hamming，因为对二进制串而言这是自然且高效的度量；SIFT 用 L2，因为浮点向量需在欧氏空间中比较方向直方图。
- **稳定性**：SIFT 通过梯度高斯加权 + 多尺度差分 (DoG) 获得高度稳定的方向与尺度估计，对光照、模糊、视角变化更鲁棒；ORB 用强度质心估计方向、用金字塔近似尺度，速度更快但稳定性稍差。
- **速度**：ORB 检测（FAST）和描述（BRIEF）都是位运算，远快于 SIFT 的浮点卷积。
- **本实验中**：SIFT 内点比例 0.9375，远高于 ORB 的 0.1812；尽管 ORB 总匹配数量更多（287 vs 80），但其中大量是错误匹配。两者都成功完成定位，**SIFT 的匹配质量更高，ORB 在原始匹配数量与速度上更具优势**。

### 问题 9：SIFT 是否抗透视变换？
- SIFT 主要抗 **平移、旋转、尺度、亮度线性变化、轻度模糊**，对一定范围内的视角变化也有较好鲁棒性。
- **SIFT 不严格抗透视变换**。SIFT 的描述子也只在关键点局部邻域内构建，对小邻域而言透视变换近似为仿射变换，所以小视角变化下仍能匹配；但大角度透视下，邻域内的梯度方向直方图会显著变形，描述子不再相似，从而失效。
- 在本实验中，**真正处理整体透视/几何变化的是 RANSAC + Homography**——SIFT/ORB 只负责给出局部对应，全局透视关系由 Homography 拟合，错误匹配由 RANSAC 剔除。

### 问题 10：实验总结（300–500 字）

本实验围绕 OpenCV 的局部特征匹配，完成了从 ORB 关键点检测、暴力匹配、RANSAC 错误匹配剔除、Homography 目标定位，到 `nfeatures` 参数对比与 SIFT 选做对比的全流程。

ORB 特征检测与匹配的基本流程为：(1) 用 `cv2.ORB_create` 在多尺度金字塔上检测 FAST 关键点并通过强度质心法估计方向；(2) 在每个关键点上用 rBRIEF 计算 256 位二进制描述子；(3) 用 BFMatcher + NORM_HAMMING + crossCheck 进行暴力匹配并按距离排序；(4) 提取匹配点对的坐标，利用 `cv2.findHomography` + `cv2.RANSAC` 估计单应矩阵并得到内点 mask；(5) 用 `cv2.perspectiveTransform` 将模板图四个角点映射到场景图，最后用 `cv2.polylines` 画出目标边框。

RANSAC 与 Homography 在目标定位中的作用是互补的：RANSAC 通过随机采样和几何一致性投票，**在大量错误匹配中找出真正满足同一变换的内点子集**；Homography 则将这些内点拟合成一个全局映射，从而把模板图坐标系下的角点准确投影到场景图。

实验中最直观的问题是初始匹配中存在大量错误匹配（ORB 内点比例仅 18%），原因是包装盒文字与图案存在重复纹理和局部相似性。增大 `nfeatures` 反而进一步压低了内点比例。SIFT 配合 Lowe ratio test 后内点比例高达 94%，但匹配数量大幅减少、速度也较慢。

由此可以得出对特征匹配鲁棒性的几点理解：**局部描述子只能保证局部光度相似性**，必须配合几何一致性（RANSAC + Homography）才能得到鲁棒的全局对应；**特征点不是越多越好**，质量比数量重要；**描述子选择需要在速度与稳定性之间权衡**——ORB 适合实时场景，SIFT 适合离线、对精度敏感的任务。
